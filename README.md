## Управление ветками

1. Для себя создаем ветку "feature/<feat_name>".
   Эту ветку в дальнейшем пушим в удаленный репозиторий и не удаляем!

2. Работу проводим либо в своей ветке, либо в дочерних (создаем по необходимости, соблюдая нейминг).
   Дочерние ветки можно удалять и можно не пушить в Gitlab;

3. Результаты работы товарища можно мерджить себе, но делайте это аккуратно! При помещении себе
   чужой работы могут произойти конфликты или еще какие ошибки. Так что, чтобы меньше мучаться, можно
   создать для мерджа себе дочернюю ветку, в ней убедиться, что всё раотает, и мерджить её в свою ветку.

4. Промежуточные коммиты называть "feature/<feat_name>: <last_changes>".

5. Коммиты стабильных версий называть "feature/<feat_name>/stableX",
   где Х - номер стабильной версии вашей ветки.

6. Компилируем исходники в файлы <filename>.out, т.к. это расширение лежит в .gitignore.
   Если скомпилили с другим названием - проследите, чтобы файл не фиксировался гитом. Не комитьте его!
